La personnalisation en détail.

Toutes les méthodes de personnalisation avec Hugo

Les Thèmes

1. Choisir et installer un thème

Hugo propose une bibliothèque de thèmes que l'on peut utiliser pour donner à un site une apparence prête à l'emploi. Le choix du thème se fait en parcourant la bibliothèque sur le site Hugo dans la rubrique Themes et en suivant les instructions du thème pour l'installer.

2. Configurer le thème

La plupart des thèmes ont un fichier de configuration (par exemple config.toml ou config.yaml) où l'on peut spécifier des paramètres spécifiques au thème. Pour connaître les options de configuration disponibles il faut consulter la documentation du thème.

Feuilles de style (CSS)

1. Ajout de styles personnalisés

Les fichiers de feuilles de style personnalisés (par exemple, styles.css) doivent être placés dans le répertoire static du projet. Hugo inclura automatiquement ces fichiers lors de la génération du site. On peut alors écrire notre propres styles ou remplacer les styles existants du thème.

2. Référencement dans les modèles

S'assurer que les modèles font référence aux fichiers CSS ajoutés. Il est possible de le faire en incluant une balise dans l'en-tête des modèles. Certains thèmes incluent déjà une référence aux fichiers de style, mais on peut personnaliser cela en fonction des besoins.

Sass/Less

Utilisation de préprocesseurs CSS

Certains thèmes Hugo peuvent utiliser des préprocesseurs CSS tels que Sass ou Less. Si c'est le cas, il est possible de personnaliser le style en modifiant les fichiers source Sass ou Less plutôt que les fichiers CSS générés. Il faut s'assurer de suivre les instructions du thème sur la manière de compiler ces fichiers dans des fichiers CSS.

Templates HTML

Personnalisation des classes et IDs

Les templates HTML dans le répertoire layouts peuvent contenir des classes et des IDs spécifiques. Ces classes et IDs sont personnalisables afin d'appliquer notre propres styles. Par exemple, dans le fichier layouts/_default/baseof.html, il est possible de modifier les balises HTML ou d'ajouter des classes supplémentaires.

JavaScript

Ajout de scripts personnalisés

L'ajout de scripts JavaScript personnalisés se fait dans le répertoire static et les référencements dans les modèles. Il faut s'assurer de respecter l'ordre d'inclusion des scripts si cela est spécifié dans la documentation du thème.

Modification des images

Personnalisation des images

Si le thème utilise des images pour les arrière-plans, les logos, etc., il est possible de les remplacer en plaçant d'autres images dans le répertoire static et en s'assurant que les modèles font référence aux nouvelles images.

Thème Enfant :

Création d'un Thème Enfant

Pour ne pas modifier directement le thème que utilisez, on peut créer un thème enfant. Cela permet de personnaliser le style sans toucher au code du thème principal.
